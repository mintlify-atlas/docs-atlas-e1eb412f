The `interactive` command provides a user-friendly, wizard-style interface for configuring and generating documentation. Instead of specifying all options via command-line flags, you can answer a series of prompts to build your configuration.

## Usage

```bash
doc-kit interactive
```

## Options

<ParamField path="-h, --help" type="boolean">
  Display help information for the interactive command.
</ParamField>

## How It Works

The interactive mode guides you through the documentation generation process with a series of prompts. This is particularly useful when:

- You're new to doc-kit and want to explore available options
- You need to configure multiple generators with different settings
- You prefer a guided experience over memorizing CLI flags
- You want to save time by avoiding manual configuration file creation

## Examples

### Basic Usage

Launch the interactive wizard:

<CodeGroup>
```bash npm
npx doc-kit interactive
```

```bash node
node bin/cli.mjs interactive
```
</CodeGroup>

```
  URL

  ◯ man-page

✓ Documentation generated successfully
```

### Quick Start Workflow

For first-time users, the interactive mode provides the fastest way to generate documentation:

1. Run `npx doc-kit interactive`
2. Follow the prompts to select your options
3. Review the configuration summary
4. Confirm to generate documentation

```bash
npx doc-kit interactive
```

## When to Use Interactive Mode

<AccordionGroup>
  <Accordion title="First-Time Setup">
    When you're setting up doc-kit for the first time, interactive mode helps you:
    - Understand what generators are available
    - Learn about required vs. optional configuration
    - Create a working configuration without reading extensive documentation
  </Accordion>
  
  <Accordion title="Complex Configurations">
    For complex documentation projects with:
    - Multiple target generators
    - Custom version and changelog settings
    - Specific threading and performance requirements
    
    Interactive mode makes it easier to configure all options correctly.
  </Accordion>
  
  <Accordion title="Quick Experiments">
    When you want to quickly test different generator combinations or settings without editing configuration files.
  </Accordion>
</AccordionGroup>

## Transitioning to CLI

Once you're familiar with the options, you can transition to using the [`generate`](/commands/generate) command directly:

```bash
# Instead of interactive prompts
npx doc-kit interactive

# Use direct CLI flags
npx doc-kit generate -t web -i "docs/**/*.md" -o out
```

Or create a configuration file for reusable settings:

```bash
npx doc-kit generate --config-file ./doc-kit.config.mjs
```

## Tips

<Tip>
  Use the arrow keys to navigate options and the spacebar to select multiple choices when prompted for generators.
</Tip>

<Tip>
  Press `Ctrl+C` at any time to exit the interactive wizard without generating documentation.
</Tip>
